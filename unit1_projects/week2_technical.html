Static vs relative vs absolute vs fixed positioning

There are many types of positioning systems one can use - and they can be utilized to really add standout content to a website or blog. The main choices we have are

  • Static positioning
  • Relative positioning
  • Absolute positioning
  • Fixed positioning

    • Static positioning is boring - it's exaclty what you would think. It's the default positioning of content on a website that comes as if you didn't have any kind of styling added to it. The only reason you would actually set an element to static is to remove some positioning that was outside your control.

    Relative positioning is a bit of a misnomer. Setting an element to position: relative; on an element with no other positioning attributes doesn't affect its position (it will be static) - but establish that element as a base by which other elements can be positioned. However, if you do give it some positioning attribute (like for example top: 10px;) it will shift its position 10 pixels down from where it would normally be. It's often used to line up form elements. Setting a relative position does two toher things:

  • it introduces the ability to use z-index on that element, which doesn't really work with statically positioned elements. This element will now appear on top of any other statically positioned eelments
  • It also limits the scope of aboslutely positioned child elements. Any element that's a child of that relatively positioned element can be absolutely positioned within that block
