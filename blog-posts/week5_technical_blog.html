Instance variables and instance methods

Instance variables are variables that can be defined for different objects within a class. It's used, oftentimes, in comparison to class variables - because class variables are the same variable defined consistently across a class. Instance variables are oftentimes used when you need different values for objects within the same class. A good example of this would be the following - let's say you define a class Polygon with 10 sides:

class Polygon
@@sides=10
def self.sides
@sides
end
end

puts Polygon.sides #=>10

However, if you create subclasses within Polygons, let's say a Triangle with 3 slides, changing the class variable sides for Triangle also changes this value for Polygons

class Triangle < Polygon
@@sides = 3
end

puts Triangle.sides # => 3
puts Polygon.sides # => 3

However, instead if we focus on specifying class level instance variables, we won't have this problem:

class Polygon
class << self; attr_accessor :sides end
@sides = 8
end

puts Polygon.sides # => 8
class Triangle < Polygon
@sides = 3
end

puts Triangle.sides # => 3
puts Polygon.sides # => 8

At the end of the day, what the instance variable allows us to do is to separate different values (3 and 8) for different objects.
